# 上班打卡小程序

一个基于 Vue 3 + IndexedDB + Element Plus 的本地化上班打卡管理应用，支持自动工资计算、数据可视化统计和日历视图。

## 功能特点

### 打卡管理
- **上下班打卡**：记录每日上下班时间，自动生成时间戳
- **自动状态判断**：根据标准时间自动判断迟到/早退/缺勤
- **补打卡功能**：支持单日和批量补打卡操作
- **记录编辑**：修改已有打卡记录（时间、状态、日薪）

### 工资计算
- **灵活配置**：可设置日薪、周末加班倍数、标准上下班时间
- **自动计算**：根据打卡时间和配置自动计算日薪
- **周末倍数**：周末工作按倍数计算工资（如双倍）
- **加班计算**：超过8小时工作按加班费计算

### 数据可视化
- **日历视图**：月度打卡情况一目了然，颜色标记不同状态
- **统计报表**：工资趋势折线图、出勤分布饼图（ECharts）
- **月度统计**：工作天数、总工资、平均日薪、加班时长等关键指标

### 数据管理
- **本地存储**：使用 IndexedDB 实现浏览器端数据持久化
- **数据备份**：支持 JSON 格式数据导出和导入恢复
- **暗黑模式**：跟随系统颜色方案自动切换深色/浅色主题

## 主要页面

### 1. 打卡主页 (`/`)
- 显示今日日期、打卡状态、工作时长、今日收入
- 上班/下班打卡按钮（含 loading 状态和 disabled 逻辑）
- 补打卡对话框（选择日期、时间、状态、手工日薪）
- 最近10条打卡记录表格（可编辑、删除）

### 2. 日历视图 (`/calendar`)
- 月度日历网格，支持上/下月切换和回到今天
- 日期颜色标记（周末、今日、有记录、当前月）
- 点击日期查看详情（上下班时间、工作时长、状态、日薪）
- 单日编辑/删除
- 批量补打卡（选择日期范围、统一设置时间和状态，含预览）

### 3. 统计报表 (`/statistics`)
- 月度选择器（最近6个月）
- 总工资、工作天数、平均日薪、加班时长概览卡片
- 工资趋势折线图（ECharts，带面积渐变）
- 出勤分布饼图（正常/迟到/早退/缺勤/加班）
- 月度详细记录表格

### 4. 设置页面 (`/settings`)
- **工资设置**：日薪、周末倍数、标准上下班时间（带表单验证）
- **工资计算说明**：规则说明和计算示例
- **数据管理**：导出 JSON 备份、导入恢复、清除记录
- **数据统计**：总记录数、总工资、最早/最近记录
- **关于**：版本信息、功能列表、技术栈、使用说明

## 配置说明

### 工资参数
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 日薪 | 正常工作日的每日工资 | 300 元 |
| 周末加班倍数 | 周末工作的工资倍数 | 2 倍 |
| 标准上班时间 | 用于判断迟到 | 09:00 |
| 标准下班时间 | 用于判断早退 | 18:00 |

### 计算规则
1. **基础日薪**：工作日按日薪，周末按 `日薪 × 周末倍数`
2. **加班工资**：超过8小时的部分，按 `小时工资 × 周末倍数`
3. **小时工资**：`日薪 ÷ 8`
4. **示例**：日薪300元，周末倍数2，工作日加班2小时
   - 基础日薪：300元
   - 加班工资：`(300 ÷ 8) × 2 × 2 = 150元`
   - 总工资：450元

## 数据库说明

使用 IndexedDB 浏览器本地存储，包含3个对象存储：

| 存储名称 | 主键 | 用途 |
|----------|------|------|
| `salary_settings` | `id` | 工资配置（单条记录，固定 id='primary'） |
| `punch_records` | `id` | 打卡记录（索引：date, month） |
| `monthly_summaries` | `id` | 月度统计摘要 |

数据完全存储在浏览器本地，不依赖网络。可通过设置页面导出为 JSON 文件备份。

## 使用说明

1. **首次使用**：前往设置页配置工资参数（日薪、倍数、上下班时间）
2. **日常打卡**：在打卡主页进行上下班打卡
3. **补打卡**：在打卡页或日历页点击”补打卡”按钮
4. **批量补打卡**：在日历页点击”批量补打卡”
5. **查看统计**：在统计页查看工资趋势和出勤情况，支持切换月份
6. **数据备份**：定期在设置页导出数据备份
7. **数据恢复**：在设置页选择 JSON 文件导入

## 已知/待处理问题

1. **IndexedDB `getByIndex` 空值保护缺失**：`src/db/index.js` 中 `getByIndex` 方法没有 `this.db` 空值检查（`getAll` 方法有但修复逻辑无效），数据库初始化失败时会导致 `Cannot read properties of null (reading 'transaction')` 报错，应用无法正常加载数据
2. **`server.log` 文件膨胀**：项目根目录下存在 124MB 的 `server.log` 文件，占用大量磁盘空间，建议定期清理
3. **`el-button type=”text”` 弃用警告**：`Home.vue` 和 `Statistics.vue` 中使用 `type=”text”`，Element Plus 3.0 将移除该属性，需改为 `type=”link”`
4. **`stores/` 目录为空**：`src/stores/` 目录下无文件，如需状态管理可引入 Pinia
